                       -- as its argument and returns an Int

Explanation:
The type of mySum tells us that mySum takes a list of Ints as an argument
and returns an Int that is the sum of all the Ints in the argument list.

the argument list is empty, for which case the function value is 
defined as zero (summation identity).

Def.2 is when the argument list contains at least one element,

---- Question 6.2 (10 points)
polynom1 :: Num a => [a] -> Int -> a -> a
polynom1 [] n x = 0
polynom1 (a:as) n x = a * term n x + polynom1 as (n-1) x
{- 

the function takes a list of coefficients [a] (like [an, an-1, ..., a0]), and int values,
and returns a num which is the value of the polynomial at x.

polynom1 [] n x = 0
This is the base case.
If the coefficient list is empty, there are no terms left to add, so the value contributed is 0.

polynom1 (a:as) n x = a * term n x + polynom1 as (n-1) x
it cercusrively recursion splits the coefficient list into the first coefficient a and the remaining coefficients as.
then the term n x computes x^n using explicit recursion and a * term n x computes the current term of the polynomial: a * x^n
the polynom1 recursively evaluates the rest of the polynomial using the remaining coefficients,
but now with degree reduced by 1, because the next coefficient corresponds to x^(n-1).
finally, we add the current term to the recursive result to build the full polynomial value
-}

---- Question 7.1 (10 points)
polynom2 :: Num a => [a] -> a -> a -> a
polynom2 [] res x = res
polynom2 (a:as) res x = polynom2 as ((res * x) + a) x
{-
polynom2 evaluates a polynomial recusrively by recomputing powers of x one by one and
the function takes a list of coefficients, a running result res which gets updated every call, and a value x
the base case is when the list is empty, in which case the result is returned
the recursive case, the function multiplies the current result by x and adds the next coefficient
then passes this new value into the next recursive call. this creates the effect of the last polynomial function
this function does NOT recompute powers of x.
instead, it reuses the previous result and multiplies by x once per coefficient,
which follows Horner’s method :)

-}
